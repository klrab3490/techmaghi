JavaScript Error Handling

try-catch

try-catch blocks are the most basic structures used for error handling in JavaScript. 

                    try {
                        // code that may throw an error
                    } catch (error) {
                        // handle the error
                    }

Asynchronous Operations

  • Promise
    • Its an object is an eventual completion of an asynchronous operation
    • .then and .catch methods are used to handle the result or error of the promise
    • It can be in one of three states: pending, fulfilled, or rejected
  • Callbacks
  • async/await
    • Syntactic sugar for promises
    • Allows writing asynchronous code in a synchronous manner
    • Built on top of promises

Example: Promise with try-catch

function fetchData() {
        return new Promise((resolve, reject) => {
            try {
                // Simulate an API call
                setTimeout(() => {
                    const data = { id: 1, name: 'John Doe' };
                    resolve(data);
                }, 1000);
            } catch (error) {
                reject(error);
            }
        });
    }

Example: async/await with try-catch

async function fetchData() {
        try {
            const response = await fetch('https://api.example.com/data');
            const data = await response.json();
            console.log(data);
        } catch (error) {
            console.error('Error:', error);
        }
    }
